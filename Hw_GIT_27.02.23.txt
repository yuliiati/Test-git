1 уровень сложности: 


1. Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

- git checkout

2. Как вернуть файл из индекса в рабочий каталог?

- git restore

3. Как отменить ошибочный commit ?

- git reset

Отмена последнего коммита:

- git reset HEAD~

Полное удаление последнего коммита:

- git reset --hard HEAD~1

4. Как создать новую ветку и переключится в нее?

- git checkout -b <название ветки> 

5. В чем отличение между fast-forward merge  and non fast-forfard merge?

Основная идея в использовании fast forward git-ом – это “не плодить лишних сущностей”, если в этом нет необходимости: если в мастер не было изменений, то при мердже из девелопа все коммиты будут в ветке мастера.

Если же изменения были в мастере, и в девелоп-ветке – то git сохранит две отдельных линии коммитов.

6. Когда может возникнуть конфликт слияния веток, как  его разрешить?

Конфликт возникает, когда в двух ветках была изменена одна и та же строка в файле или когда некий файл удален в одной ветке и отредактирован в другой. Как правило, конфликты возникают при работе в команде.

Существует множество способов разрешения конфликтов слияния. 

Например, такие команды git log, git reset, git status, git checkout и git reset. 

Инструменты для случаев, когда Git прерывает работу в самом начале слияния:

- git checkout

Команда checkout может использоваться для отмены изменений в файлах или для изменения веток.

- git reset --mixed

Команда reset может использоваться для отмены изменений в рабочем каталоге или в разделе проиндексированных файлов.


Инструменты для случаев, когда конфликты Git возникают во время слияния:


- git merge --abort

При выполнении команды git merge с опцией --abort процесс слияния будет прерван, а ветка вернется к состоянию, в котором она находилась до начала слияния.

- git reset

Команду git reset можно использовать для разрешения конфликтов, возникающих во время выполнения слияния, чтобы восстановить заведомо удовлетворительное состояние конфликтующих файлов.


7. Можно ли удалить ветку? Как? 

В локальном репо:

- git branch -d  local_branch_name

В удаленном репо:

- git push remote_name -d remote_branch_name
